# OWN Acquiring APIs - Authentication

## Overview

To use the OWN integration resources and solutions, authentication must be performed using the OAuth 2.0 protocol.

This document describes how to authenticate and retrieve an access token for consuming OWN Acquiring APIs.

---

## Authentication Endpoints

### Sandbox

```txt
https://acquirer-qa.own.financial/agilli/v2/auth
```

### Production

```txt
https://acquirer.own.financial/agilli/v2/auth
```

---

## Definitions

| Term | Description |
|---|---|
| Client | Company (CPF or CNPJ) that has a contractual relationship with OWN Financial, such as a white-label client. |
| Partner | Company (CPF or CNPJ) that has a contractual relationship with both the OWN Financial client and the supplier, such as gas stations, restaurants, stores, or other final establishments. |

---

## Authentication Flow

### HTTP Method

```http
POST
```

---

## Request Payload

Send the following JSON payload to the authentication endpoint:

```json
{
  "client_id": "xxxxxxxxxxxxx-api-acquirer.wl",
  "client_secret": "xxxxxxxxxxxxxxx",
  "scope": "xxxxxxxxxxx",
  "grant_type": "client_credentials"
}
```

---

## Request Parameters

| Field | Description |
|---|---|
| `client_id` | Your OWN client identifier. |
| `client_secret` | Secret key generated by OWN. |
| `scope` | Integration scope allowed for the client. |
| `grant_type` | Token authentication mechanism. Use `client_credentials`. |

After completing the OWN client registration process, the credentials will be provided by email.

---

## Response Payload

Successful authentication returns a JSON response similar to the following:

```json
{
  "access_token": "xxxxxxxxxxxxxxxx",
  "expires_in": 300,
  "refresh_expires_in": 0,
  "token_type": "Bearer",
  "not-before-policy": 0,
  "scope": "own.api_wl.api"
}
```

---

## Response Fields

| Field | Description |
|---|---|
| `access_token` | Access token used to consume other API endpoints. |
| `expires_in` | Token expiration time in seconds. |
| `refresh_expires_in` | Refresh token expiration time in seconds. |
| `token_type` | Type of token returned. |
| `not-before-policy` | Informational security policy identifier. |
| `scope` | Integration scopes enabled for the token. |

---

## Example Request

### cURL

```bash
curl --request POST \
  --url https://acquirer-qa.own.financial/agilli/v2/auth \
  --header 'Content-Type: application/json' \
  --data '{
    "client_id": "YOUR_CLIENT_ID",
    "client_secret": "YOUR_CLIENT_SECRET",
    "scope": "YOUR_SCOPE",
    "grant_type": "client_credentials"
  }'
```

---

## Using the Access Token

The generated token must be sent as a Bearer token in the `Authorization` header when consuming other OWN API endpoints.

### Example

```http
Authorization: Bearer YOUR_ACCESS_TOKEN
```

---

## LLM Optimization Notes

- Authentication uses OAuth 2.0 Client Credentials flow.
- Tokens are obtained through a POST request to `/auth`.
- `grant_type` must always be `client_credentials`.
- The returned `access_token` is required for all subsequent API requests.
- Tokens expire according to the `expires_in` value.
- Authentication environments:
  - Sandbox: `acquirer-qa.own.financial`
  - Production: `acquirer.own.financial`
